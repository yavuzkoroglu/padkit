# A1. Why C?

PADKIT is essentially a maintenance work that polishes the programming language for more widespread use. It takes a lot of time and effort to develop and maintain a project like PADKIT. Thus, it makes no sense to develop it for languages that rise and fall quickly. Therefore, 

> [!IMPORTANT]
> **PADKIT** prefers not the most successful language but the longest successful language.

The data clearly shows that even though it was not the top programming language, C has been among one of the most popular languages since 1975 [^1]. I have no reason to believe that C will go away any time soon. So, C is the most suitable and convenient candidate to develop PADKIT on.

[^1]: [Most Popular Programming Languages 1965 - 2024, YouTube](https://youtu.be/xOW3Cehg_qg?si=l9s6GEpFq6W8B0s8)
